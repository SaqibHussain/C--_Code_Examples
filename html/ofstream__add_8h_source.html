6 // Short function to resolve the problem of implementing the
7 // friend ofstream operator of a class with a conversion
8 // constructor (ie. a non-explicit constructor with one parameter).
9 //
10 // Problem in detail:
11 // 1. The ofstream class does not have the operator<< implemented
12 // 2. The ofstream is a derived class of the ostream, which has
13 // the operator<<(double) implemented
14 // 3. In order to call the base class operator, the implicit cast
15 // must be done from ofstream& to ostream&
16 // 4. The standard conversion from Derived& to Base& has the same
17 // priority as some other conversions (ex. double to int) in
18 // the overloaded function ambiguity resolve mechanism.
19 // 5. Since the priority is the same, we end up with ambiguous
20 // call, compiler doesn't know what we want to call.
21 //
22 // Solution:
23 // We introduce a generic template function which provides the
24 // explicit cast, and works as a 'proxy'. The calls are no longer
25 // ambigous, because if no-match exists for the specialized
26 // ofstream operator<<, then the function below is used.
27 // Ie. it resolves ambiguity of a call by favouring the ofstream
28 // to ostream upcast (and not permitting the implicit conversion
29 // of a T type if there is a proper ostream operator<<)
30 //
31 
32 template <typename T>
33 std::ofstream& operator<<(std::ofstream& ofs, const T& val)
34 {
35  static_cast<std::ostream&>(ofs) << val;
36  return ofs;
37 }
